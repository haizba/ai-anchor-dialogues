# 人工智能主播对话 / AI Anchor Dialogues

这是一个人与 AI 长期对话生成思想的原始记录。

这个仓库不是一个成熟代码项目，也不是一个标准工程教程。  
它记录的是：一个操作者如何通过提示词、文档、终端反馈、本地 AI、APK、加固壳和 AI 对话，逐渐形成一套“锚点—回声—添加—身份流—操作流”的方法。

## 核心问题

我关心的不是 AI 单次回答是否正确，而是：

- 提示词如何成为锚点
- 文档如何成为记忆层
- 人如何纠正 AI 的线性化倾向
- 本地小模型如何成为操作代理
- 本地小模型如何进入身份流
- APK / 加固壳 / trace 如何成为标准锚点
- 人、AI、文档、终端、系统如何共同生成因果图谱

## 当前关键词

- 添加
- 提示词锚点
- 操作折返
- 身份折返

- APK / 壳标准锚点
- 后验因果图谱
